MCP servers

Take the document search you built in Lesson 1 and expose it as a Model Context Protocol tool — so Claude Code can query your documents/ folder natively, no copy-paste.

What is MCP, and why?

The Model Context Protocol (MCP) is an open standard that lets an AI client (like Claude Code) discover and call external tools over a simple JSON-RPC connection. Instead of you pasting document text into a prompt, the model calls a search_docs tool and pulls the context itself.

Lesson 1 vs Lesson 2: in Lesson 1 your app drove the pipeline and called the LLM. In Lesson 2 the relationship flips — the LLM drives, and your retriever becomes a tool it reaches for on demand. Same retrieval engine, new integration surface.

The shape of what we'll build

A tiny stdio MCP server that wraps the Lesson 1 retriever and advertises one tool:

mcp_server.py (sketch)
from mcp.server import Server
from localrag.config import load_config
from localrag.engine import get_retriever

server = Server("local-ai-lab-docs")

@server.tool()
def search_docs(query: str, k: int = 5) -> str:
    """Search the user's local documents and return the most relevant passages."""
    config = load_config()
    hits = get_retriever(config).search(query, k)
    return "\n\n".join(f"[{h['source']}:{h['page_number']}] {h['text']}" for h in hits)

# run over stdio so Claude Code can launch it as a subprocess
Notice the reuse: get_retriever and the chunk format come straight from Lesson 1. MCP is just a new doorway onto the same engine.
